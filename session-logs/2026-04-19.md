## Technical Learnings
- Supabase MCP cannot change Auth provider settings (email confirmation toggle) — must be done in dashboard manually
- `DEFAULT auth.uid()` on the column means inserts from authenticated sessions auto-populate `user_id` without changing any query code
- Data migration needed for existing rows after adding `user_id` — existing rows had `user_id IS NULL` and were invisible under new RLS until `UPDATE` was run
- PR check must read `getBestWeightForEx` BEFORE `upsertLog` — reading after includes the new record in the "previous best" query

## Decisions Made
- Email confirmation disabled (no verification flow needed for personal app)
- Custom program stays in localStorage (per-device) — not user-scoped in DB
- `user_id` column nullable initially so migration could be run safely before linking existing rows
