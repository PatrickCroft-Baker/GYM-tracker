## Technical Learnings
- Supabase JSONB for sets array allows flexible storage without schema changes
- Async/await critical for cloud data — UI must show loading states during fetch
- RLS policies even for single-user apps provide security baseline
- CSV export requires flattening JSONB array rows for proper columnar output

## Decisions Made
- Kept drafts/session state in localStorage (temporary UI state, not permanent data) — speeds up UX
- Used hardcoded anon key (acceptable for personal app with no sensitive data)
- No user authentication — public insert/read via anon key
- CSV export triggers download directly in browser (no server processing)
